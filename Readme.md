# URI / Beecrowd Solutions in C

This repository contains accepted solutions for problems from the Beecrowd (formerly URI Online Judge) platform written in C.

The goal is to provide a simple collection of solutions that students and programming enthusiasts can use to study algorithms, compare different approaches, and improve their problem-solving skills.

Feel free to explore the repository and contribute your own accepted solutions.

## Repository Structure

Solutions are organized according to the official Beecrowd categories.

```
1.BEGINNER/
    1038.c
    1074.c
    ...

2.AD-HOC/
    1121.c
    1129.c
    ...

3.STRINGS/
    1024.c
    1262.c
    ...
```

Each source file must be named using the problem number:

```
1038.c
1074.c
1121.c
```

## Source File Header

Every solution should start with the following header:

```c
/**
 * @author Your Name
 * @brief Frequency counting using a 128x128 matrix. (optional)
 * @version 0.1 (optional)
 *
 * @problem: 0000 - title
 */
```

The header should include:

- `@author`: The author of the solution.
- `@brief`: An optional short description of the solution or approach.
- `@version`: Optional. Use it to track improvements, fixes, optimizations, or community contributions to the solution over time.
- `@problem`: The Beecrowd problem number and official title.

## Contribution Guidelines

Before submitting a solution, please make sure that:

- The solution is written in **C**.
- The file name matches the problem number (e.g. `1737.c`).
- The solution is placed inside the correct Beecrowd category.
- The solution has an **Accepted** status on Beecrowd.
- The source file includes the required header.

Contributions following these guidelines are welcome.
